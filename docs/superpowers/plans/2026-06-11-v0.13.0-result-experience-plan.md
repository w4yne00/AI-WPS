# v0.13.0 Result Experience Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `v0.13.0-alpha` with adapter autostart packaging, stronger Smart Write read-only result visualization, and a Document Review issue-handling loop without changing writeback behavior or backend task routing.

**Architecture:** Keep the three backend task contracts stable and implement the new behavior in the adapter start kit and task-pane presentation layer. Smart Write preview gains a read-only display model derived from existing result text and original text; Document Review gains front-end issue state and review-record generation based on existing parsed review data. Existing copy/apply/writeback functions remain callable through their current paths.

**Tech Stack:** WPS JS task pane, plain JavaScript helpers, safe Markdown rendering, adapter start-kit Bash scripts, Python unittest packaging checks.

- [ ] **Step 3: Implement pure review record helper**

Add:

```javascript
function buildDocumentReviewRecord(data, statusByIndex) {
  var issues = data && data.issues ? data.issues : [];
  var statusText = { pending: "待处理", done: "已处理", ignored: "已忽略" };
  var categoryText = { typo: "错别字", expression: "语言表达", logic: "逻辑表达", fluency: "通畅性", professional: "专业性" };
  var counts = { pending: 0, done: 0, ignored: 0 };
  issues.forEach(function (_, index) {
    var status = statusByIndex && statusByIndex[String(index)] ? statusByIndex[String(index)] : "pending";
    counts[status] = (counts[status] || 0) + 1;
  });
  var lines = ["文档审查处理记录", "", "## 处理概览", "", "- 问题总数：" + issues.length, "- 待处理：" + counts.pending, "- 已处理：" + counts.done, "- 已忽略：" + counts.ignored, ""];
  issues.forEach(function (issue, index) {
    var status = statusByIndex && statusByIndex[String(index)] ? statusByIndex[String(index)] : "pending";
    lines.push("### " + (index + 1) + ". " + (categoryText[issue.category] || "其他问题") + "（" + statusText[status] + "）");
    lines.push("- 位置：" + (issue.location || "未标注"));
    lines.push("- 原文：" + (issue.originalText || "未提供"));
    lines.push("- 问题：" + (issue.problem || "未提供"));
    lines.push("- 建议：" + (issue.suggestion || "未提供"));
    if (issue.suggestedRewrite) {
      lines.push("- 建议改写：" + issue.suggestedRewrite);
    }
    lines.push("");
  });
  return lines.join("\n").trim();
}
```
