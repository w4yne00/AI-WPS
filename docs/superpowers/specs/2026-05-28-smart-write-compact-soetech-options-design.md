# Smart Write Compact SOE Technical Options Design

日期：2026-05-28

目标版本：`v0.12.3-alpha`

## 背景

当前智能编写任务窗口的设置区占用偏高：四个下拉项在窄侧边栏内堆叠，常驻提示词明细卡重复展示信息，导致 Markdown 结果预览空间不足。用户希望在保持苹果式蓝灰白简约风格的基础上，扩大结果预览，并将“表达风格、侧重点、篇幅”选项改得更贴近国企技术方案和汇报材料。

## 设计结论

采用方案 A：紧凑设置区 + 扩大结果预览。

- 前台只调整智能编写交互布局、菜单名称和提示词文案，不改变任务调用接口。
- 隐藏常驻提示词明细卡，保留一行“当前要求”摘要。
- 智能编写模式下提高 `#result-output` 最小高度。
- 前端只展示新选项，adapter 保留旧值别名，避免历史 payload 或旧前端缓存造成不可识别。

## 菜单定义

表达风格：

- `standard`：技术方案正式。采用国企技术方案常用的正式、准确、克制表达，术语统一，避免口语化和夸张表述。
- `structured`：条理化说明。按“背景、问题、措施、结论”组织内容，强化层级、逻辑连接和可执行表述。
- `reporting`：汇报材料风格。采用汇报材料表达，先给结论，再说明进展、问题、风险和下一步安排，语言稳健。

侧重点：

- `complete`：保持信息完整。保留原文关键信息、事实、条件和约束，不遗漏责任、时间、对象和结论。
- `conclusion_risk`：结论与风险。优先突出核心结论、关键判断、主要风险、影响范围和需要关注的问题。
- `plan_next`：措施与计划。优先突出解决措施、实施路径、责任分工、时间节点和下一步安排。
- `acceptance`：验收与闭环。优先突出交付物、验收标准、问题闭环、证据材料和后续跟踪要求。

篇幅：

- `same`：保持篇幅。保持与原文相近的篇幅，只优化措辞、结构和信息组织。
- `concise`：精简。压缩冗余表达，保留关键信息和必要限定，输出更短更直接的版本。
- `expanded`：适度扩写。在不编造事实的前提下补足必要背景、逻辑衔接、措施说明和结论表达。

## 兼容要求

adapter 继续兼容旧值：

- `default`、`formal` -> `standard`
- `conclusion`、`risk` -> `conclusion_risk`
- `next_step`、`implementation` -> `plan_next`
- `default` 篇幅 -> `same`

## 不做事项

- 不调整 adapter 路由、Dify 请求体、任务级 API Key 或智能排版逻辑。
- 不改变 Markdown 渲染器的安全策略。
- 不新增依赖。
