# Word/Excel/PPT 工作流设置交互优化设计

日期：2026-07-15

状态：已完成方案确认，待用户审阅书面设计

## 1. 目标

优化 Word、Excel、PPT 三个 WPS 插件的工作流配置体验，解决以下问题：

- 统一 API Key 与工作流独立 Key 同时出现，配置概念重复。
- 所有工作流的名称、备注、Key 输入框和操作按钮同时展开，设置页过长。
- 日常工作流切换仍需“下拉选择 + 点击切换”两步。
- Word 四类任务在同一设置页连续铺开，不利于定位当前功能。

本设计只调整三宿主前台任务窗格的设置与工作流切换体验。模型请求、长任务轮询、结果预览、复制、Word 回写以及 adapter 的旧配置兼容逻辑保持不变。

## 2. 已确认决策

1. Word 设置页使用“智能编写 / 智能仿写 / 文档审查 / 格式审查”功能标签，每次只显示一类工作流。
2. Excel 设置页只显示智能分析工作流，PPT 设置页只显示智能总结工作流。
3. 任务页工作流下拉菜单选中后立即切换，移除额外“切换”按钮。
4. 三端设置页移除统一 API Key 输入区，仅保留统一 API URL。
5. adapter 继续读取统一 API Key，并继续在覆盖安装时保护该配置，保证旧部署兼容。
6. 工作流设置首页使用紧凑列表；新建和编辑进入全宽设置子页面，不使用居中弹窗或全部展开卡片。
7. 当前正在使用的工作流不得直接删除，用户必须先切换至另一个可用工作流。

## 3. 信息架构

### 3.1 设置首页

设置首页按以下顺序组织：

1. 标题栏：返回按钮、设置标题、本地 adapter 连接状态。
2. 模型服务：当前 API URL 摘要和“修改”入口。
3. 功能定位：Word 显示四个功能标签；Excel 和 PPT 不显示多余标签。
4. 工作流区域：工作流数量、新建入口、紧凑工作流列表。

API URL 默认以单行摘要展示。点击“修改”后在原位置展开 URL 输入框以及“取消 / 保存”操作。保存成功反馈“API URL 已保存”。标题栏“已连接”只表示本地 adapter 可连接，不表示模型后台 URL 或 Key 已通过业务调用验证。

Word 从某个任务功能进入设置时，自动选中对应功能标签。通过 Ribbon 直接打开设置时，默认选中智能编写。Excel 和 PPT 分别固定管理本宿主唯一任务类型。

### 3.2 工作流列表

每个工作流列表行只显示：

- 工作流名称。
- 单行备注，超出宽度时省略。
- “当前”或“Key 未配置”等状态。
- 编辑入口。
- 非当前工作流的更多操作，包括“设为当前”和“删除”。

列表不直接显示名称输入框、备注输入框或 Key 输入框。当前工作流的删除操作禁用，并说明“请先切换到其他工作流”。空列表显示明确的空状态和“新建工作流”主操作，不显示无意义的空表单。

### 3.3 新建/编辑子页面

点击“新建工作流”或编辑入口后，设置首页切换为全宽表单子页面。子页面顶部提供返回入口，底部固定“取消 / 保存”操作。

表单字段：

- 工作流名称：必填，最长 40 字，同一任务类型内不允许重名。
- 工作流备注：选填，最长 200 字。
- API Key：新建时必填；编辑时选填，留空表示保持原 Key。

编辑页面只显示“已配置 Key / 未配置 Key”，不从 adapter 读取或回显完整旧 Key。用户输入新 Key 时可临时切换明文/密文显示，该显示只作用于当前输入框。

新建页面提供“保存并设为当前”选项。当前任务类型没有任何工作流时默认勾选；已有可用工作流时默认不勾选。编辑当前工作流不会改变其当前状态。

存在未保存修改时，返回设置首页或关闭设置页必须确认。保存期间禁用返回、取消和重复提交。

## 4. 任务页工作流切换

任务页下拉菜单只读取当前任务类型的工作流：

- Word 随智能编写、智能仿写、文档审查、格式审查模式切换任务类型。
- Excel 固定使用 `excel.analysis`。
- PPT 固定使用 `ppt.slide_assistant`。

用户选择不同工作流后立即调用激活接口，不再要求点击第二个“切换”按钮。切换成功显示短暂反馈“已切换至：{工作流名称}”。切换失败时恢复原选项并显示中文错误原因。

缺少 Key 的历史工作流继续显示为“未配置 Key”，但不可激活。任务类型没有可用工作流时，下拉区域显示“未配置工作流”，并提供进入设置的入口。

模型任务提交和运行期间禁用下拉菜单。已提交任务继续使用提交时取得的认证快照；工作流切换只影响下一次任务。进入设置和返回任务页不得清空用户输入、结果预览、本地审查状态或 Word 回写按钮状态。

## 5. 三宿主隔离与视觉一致性

三端复用同一交互合同，但继续保留现有独立插件目录和任务类型边界，不增加跨宿主运行时依赖：

- Word：蓝色主题，管理四个 Word 任务类型。
- Excel：绿色主题，只管理智能分析。
- PPT：橙色主题，只管理智能总结。

设置首页采用单列布局，不使用包含多个输入区的嵌套卡片。工作流列表行保持稳定高度，操作图标使用既有图标库或熟悉符号并提供悬停说明。所有状态、校验和错误信息优先使用中文。

## 6. 数据与接口设计

继续使用现有工作流档案接口：

```text
GET    /provider/workflow-profiles?taskType={taskType}
POST   /provider/workflow-profiles
PATCH  /provider/workflow-profiles/{profileId}
POST   /provider/workflow-profiles/{profileId}/api-key
POST   /provider/workflow-profiles/{profileId}/activate
DELETE /provider/workflow-profiles/{profileId}
POST   /provider/base-url
```

三端不再绑定统一 Key 的保存、清除控件，但 adapter 继续保留对应接口、统一 Key 文件、认证兜底和安装保护。

编辑保存按以下顺序执行：

1. 前端校验名称、备注和新 Key。
2. 调用 `PATCH` 保存名称和备注。
3. 仅当用户输入新 Key 时，再调用独立 Key 接口。
4. 重新读取服务端工作流列表，以服务端状态为准刷新页面。

如果第 2 步失败，不执行 Key 替换。如果名称和备注保存成功但 Key 替换失败，明确提示“名称和备注已保存，但 Key 更换失败；原 Key 保持不变”，不得笼统显示全部失败。

删除确认必须显示将要删除的工作流名称，并说明仅删除该任务类型下的档案及对应 Key。前端保护不能替代 adapter 的当前工作流删除保护，服务端仍需拒绝删除当前档案。

## 7. 兼容与保护边界

- 不修改已有工作流档案 ID、名称、备注、Key 文件或当前选择。
- 不迁移或清空旧统一 Key。
- 覆盖安装继续保护 API URL、统一 Key、任务级 Key、工作流档案和当前选择。
- 不修改模型后台 `/chat-messages` 请求、think 内容剥离或输入模式兼容。
- 不修改文档审查、Excel 智能分析和 PPT 智能总结的长任务等待机制。
- 不修改智能编写预览、对照、复制和写回行为。
- 不修改智能仿写、格式审查、文档审查、Excel 分析和 PPT 总结的结果结构。

## 8. 错误与状态反馈

- 工作流读取失败：保留当前功能标签，显示重试入口，不渲染虚假空列表。
- 创建或修改重名：定位名称字段并显示“该功能下已存在同名工作流”。
- Key 缺失：新建时阻止保存；历史档案在列表和下拉菜单中标记。
- 激活失败：恢复原选择，不改变当前状态标记。
- 删除失败：保留列表项和当前页面，不提前做乐观删除。
- adapter 暂时不可达：保留当前表单内容，允许重试，不清空 Key 输入。
- 部分保存成功：准确说明成功和失败的字段，并重新加载服务端状态。

## 9. 测试与验收

### 9.1 自动化测试

- Word 四个任务标签只加载各自 `taskType` 的档案。
- Excel 和 PPT 不读取或显示其他宿主档案。
- 新建、修改名称、修改备注、更换 Key和删除非当前档案成功。
- 当前档案在前端和 adapter 两层均无法删除。
- 编辑时 Key 留空不会调用 Key 替换接口。
- 即时切换成功后更新当前选项；失败后恢复原选项。
- 忙碌状态禁止切换，任务认证快照不受后续选择影响。
- 统一 Key 控件不再出现在三端 HTML，但 adapter 兼容接口与安装保护测试继续通过。
- 设置往返保持 Word 四种模式、Excel 分析和 PPT 总结的输入及结果状态。

### 9.2 视觉与交互验证

在 Word、Excel、PPT 分别检查：

- 设置首页空状态、单工作流、多工作流、Key 缺失和读取失败状态。
- 新建、编辑、未保存返回确认、删除确认和保存失败状态。
- `420×900` 及更窄任务窗格无横向滚动、按钮重叠、文本溢出或底部操作遮挡。
- Word 蓝、Excel 绿、PPT 橙的宿主主题正确，布局和状态文案一致。
- 任务页即时切换反馈清楚，设置快捷入口和返回路径正确。

### 9.3 目标机回归

在麒麟 V10/WPS 目标机执行覆盖安装，确认旧 API URL、统一 Key、所有工作流档案 Key 和当前选择均保留。分别在 Word、Excel、PPT 中切换两个工作流并提交真实模型任务，核对 adapter 诊断中的 `taskType` 和认证来源命中所选档案。

## 10. 非目标

- 不新增统一任务中心或历史记录。
- 不增加工作流导入、导出、排序、批量删除或跨宿主复制。
- 不展示、复制或导出已保存的完整 Key。
- 不修改 Ribbon 入口、模型能力、结果预览内容或 Word 回写功能。
