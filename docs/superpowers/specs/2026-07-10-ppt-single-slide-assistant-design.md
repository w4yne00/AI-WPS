# AI-WPS PPT 单页内容助手设计

日期：2026-07-10

目标版本：`v0.17.0-alpha`

基线版本：`v0.16.0-alpha`

设计状态：已确认

## 1. 背景与决策

目标机上的 Dify 工作流受上下文和 max token 限制，不适合一次提交长文档、多个工作表或整份演示文稿。下一阶段功能应继续采用短输入、局部处理和可恢复后台任务，不把完整 Office 文件直接发送给模型后台。

近期规划作如下调整：

- 文档审查以用户框选的小段内容为主要使用方式，暂不建设跨全文自动定位原文。结果继续显示原文片段和选区内段落序号。
- Excel 多区域、多工作表原始数据直传暂缓。后续如需建设，应先由 adapter 在本地完成统计、差异提取和数据压缩，再把摘要交给模型解释。
- 统一任务中心、历史记录和格式审查确定性修复不进入近期版本。
- `v0.17.0-alpha` 以 PPT 单页内容助手为唯一主线。

## 2. 目标与非目标

### 2.1 目标

PPT 单页内容助手面向当前幻灯片提供两种自动识别的处理方式：

- 当前页已有有效正文时，优化标题、压缩要点并提炼结论。
- 当前页内容不足时，根据用户填写的主题和要求生成本页内容。

结果固定包含：

- 一个建议标题。
- 3 至 5 条核心要点。
- 一句本页结论。

首版只提供预览、纯文本和复制能力，不修改幻灯片。

### 2.2 非目标

首版不建设以下能力：

- 不读取或分析整份 PPT。
- 不生成完整 PPT 大纲或多页内容。
- 不读取备注、母版、隐藏页、图片内容、图表数据或嵌入对象。
- 不生成图片、图表、版式或动画。
- 不创建、删除或调整文本框和占位符。
- 不写回标题、正文或备注。
- 不提供原文对照视图。
- 不跨 Word、Excel、PPT 自动搬运内容。

## 3. 宿主与总体架构

新增独立 PPT 插件目录：

```text
formal-plugin-kit/wps-ai-assistant-wpp_1.0.0/
```

统一安装包中的 `publish.xml` 增加 WPS 演示宿主注册项，插件类型使用 `wpp`。三个宿主严格隔离：

- Word 插件只显示智能编写、智能仿写、文档审查、格式审查和设置。
- Excel 插件只显示 Excel 智能分析和设置。
- PPT 插件只显示 PPT 单页助手和设置。

adapter 新增任务类型：

```text
ppt.slide_assistant
```

PPT 功能复用以下稳定基础能力：

- Dify `/chat-messages` 新旧用户输入参数兼容。
- 任务级工作流配置档案和 API Key 隔离。
- `<think>` 内容剥离，只展示最终答案。
- 后台任务、短请求轮询、`clientJobId` 幂等和任务恢复。
- provider 脱敏诊断和非标准结果兜底。
- 安装升级时保护现场 API URL、API Key 和工作流档案。

Word 和 Excel 的现有路由、前端功能、读取逻辑、模型提示词和写回边界不改。

## 4. WPS 演示宿主准入验证

完整开发前必须先制作最小 `wpp` 宿主探针，并在麒麟 V10、WPS 12.1.2 目标机验证：

1. `wpp` 类型插件能被 WPS 演示识别并显示 Ribbon。
2. 能取得当前演示文稿和当前幻灯片。
3. 能读取当前页序号。
4. 能枚举当前页文本形状并读取文本。
5. 能识别标题占位符；无法识别时能使用安全的标题回退规则。
6. 能只读取前一页和后一页的标题。
7. 空白页、仅图片页、图表页和没有标题占位符的页面不会导致任务窗格卡死。

宿主探针未通过时，不进入模型接口和完整任务窗格开发。兼容问题必须在真实目标机上解决，不使用模拟数据掩盖 WPS 对象模型差异。

## 5. 任务窗格设计

### 5.1 Ribbon

PPT Ribbon 只提供：

- `PPT 单页助手`
- `设置`

按钮图标、尺寸和交互方式与现有 Word、Excel 插件保持一致。

### 5.2 主流程

任务窗格按以下顺序组织：

1. 当前工作流配置下拉菜单。
2. 当前页摘要。
3. 补充要求输入框。
4. `生成本页内容` 主按钮。
5. 结果预览区。

当前页摘要显示：

- 当前页序号。
- 读取到的当前页标题。
- 当前页正文字数。
- 是否读取到前一页和后一页标题。
- 是否因输入预算发生截断。

补充要求允许用户填写本页主题、汇报对象、强调重点或表达风格。用户不需要手动选择生成或优化模式。

### 5.3 结果区

结果区只提供：

- `预览`
- `纯文本`

预览固定分为：

- 建议标题。
- 核心要点。
- 本页结论。

复制操作包括：

- 复制标题。
- 复制全部要点。
- 复制结论。
- 复制完整结果。

任务窗格中不显示应用、替换、插入、写入或自动排版按钮。

## 6. 幻灯片读取与输入预算

### 6.1 读取范围

用户点击生成后，前端先刷新为“正在读取当前页”，再异步读取 WPS 对象，避免同步枚举形状造成界面无反馈。

读取范围严格限制为：

- 当前页标题。
- 当前页可读取的普通文本形状。
- 前一页标题。
- 后一页标题。

不读取备注、母版、隐藏页、图片 OCR、图表数据和整份演示文稿。

当前页标题优先取标题占位符。没有标题占位符时，使用当前页第一个非空短文本形状作为候选标题；该候选仍保留在正文形状列表中，adapter 负责去重。

当前页正文按 WPS 形状集合的稳定枚举顺序拼接。每个文本形状保留独立分隔，避免多个文本框被合并成一个连续段落。

### 6.2 预算

前端和 adapter 均执行相同的上限校验：

- 当前页标题：最多 200 个字符。
- 当前页正文合计：最多 3000 个字符。
- 单个文本形状：最多 1000 个字符。
- 前一页标题：最多 200 个字符。
- 后一页标题：最多 200 个字符。
- 用户补充要求：最多 1000 个字符。

动态业务输入总量不得超过 4600 个字符。超限时按当前页文本形状的稳定顺序截断，不继续读取后续文本形状，并在当前页摘要中显示“已按本页内容上限读取”。

这些限制集中定义为 PPT 专用常量，后续可根据现场 Dify 配额调整，不与 Word、Excel 的输入预算共用。

## 7. 生成与优化模式

模式由 adapter 统一判定，前端只发送读取结果：

- 去除标题后的当前页正文达到 20 个非空白字符时，使用 `optimize`。
- 当前页正文不足 20 个非空白字符时，使用 `generate`。
- `generate` 模式下用户补充要求不能为空。
- `optimize` 模式允许补充要求为空。

标题本身不视为有效正文，避免只有页面标题时误进入优化模式。

前后页标题只用于减少内容重复和改善衔接，不允许模型扩写前后页，也不在结果中复述相邻页内容。

## 8. adapter 接口与数据结构

PPT 单页助手只提供后台任务接口：

```text
POST /ppt/slide-assistant/jobs
GET  /ppt/slide-assistant/jobs/{jobId}
```

提交请求的业务字段包括：

```json
{
  "clientJobId": "ppt-slide-uuid",
  "slide": {
    "index": 3,
    "title": "项目进展",
    "textBlocks": ["已完成总体方案设计", "当前进入联调阶段"],
    "previousTitle": "项目背景",
    "nextTitle": "风险与措施",
    "truncated": false
  },
  "userInstruction": "面向管理层，突出当前进展和风险"
}
```

adapter 返回的完成态业务结果包括：

```json
{
  "modeUsed": "optimize",
  "suggestedTitle": "项目总体进展",
  "bullets": [
    "总体方案设计已完成",
    "系统已进入联调验证阶段",
    "重点关注跨系统接口稳定性"
  ],
  "conclusion": "项目按计划推进，下一阶段应集中完成接口联调和风险收敛。",
  "plainText": "项目总体进展\n\n1. 总体方案设计已完成\n2. 系统已进入联调验证阶段\n3. 重点关注跨系统接口稳定性\n\n项目按计划推进，下一阶段应集中完成接口联调和风险收敛。",
  "rawAnswer": null,
  "parseFallbackReason": null
}
```

任务外层状态继续使用现有后台任务语义：`running`、`completed` 和 `failed`，并返回任务号、运行耗时、provider 等待预算和可读错误摘要。

同一 `clientJobId` 只能发起一次模型任务。重复提交必须返回现有任务状态，不得重复调用模型后台。

## 9. 模型输出与解析

模型默认输出简单 Markdown：

```markdown
## 建议标题
项目总体进展

## 核心要点
- 总体方案设计已完成
- 系统已进入联调验证阶段
- 重点关注跨系统接口稳定性

## 本页结论
项目按计划推进，下一阶段应集中完成接口联调和风险收敛。
```

输出约束：

- 标题只输出一条，建议不超过 40 个中文字符。
- 核心要点输出 3 至 5 条，每条建议不超过 80 个中文字符。
- 本页结论只输出一句，建议不超过 100 个中文字符。
- 不输出页面设计说明、图标建议、配色建议、动画建议和深度思考过程。
- 不编造相邻页未提供的内容。

adapter 同时兼容标准 JSON 和上述 Markdown，并统一转换为结构化结果。解析顺序为：

1. 清理 `<think>` 内容并取得最终答案。
2. 尝试解析标准 JSON。
3. 尝试解析固定 Markdown 标题。
4. 无法结构化时保留 `rawAnswer`，并记录 `parseFallbackReason`。

非标准模型回复必须在前台可见，不能因为解析失败导致结果区空白。

## 10. 长任务与恢复

PPT 单页助手复用文档审查和 Excel 智能分析的长任务策略：

- 前端生成并持久化 `clientJobId`。
- 提交任务后使用短请求轮询状态。
- adapter 在后台继续等待模型，不让任务窗格保持单次长连接。
- 状态查询短暂失败时保留任务号并低频恢复。
- 关闭并重新打开 PPT 任务窗格后继续查询未完成任务。
- 任务成功、明确失败或超过总恢复预算后才清理本地任务号。

PPT 任务使用独立等待配置，但首版默认与 Excel 智能分析、文档审查保持同等级的慢模型预算，避免 think 模式再次出现前台先超时的问题。

## 11. 工作流配置与安全

工作流配置管理新增 `ppt.slide_assistant`：

- 每个 PPT 单页助手工作流可保存自定义名称、API Key 和备注。
- PPT 功能页可通过下拉菜单切换当前工作流。
- PPT 设置页只显示 PPT 工作流档案。
- Word 设置页仍只显示四类 Word 档案。
- Excel 设置页仍只显示 Excel 智能分析档案。
- API Key 正文继续只保存在 `run/provider_api_keys/`，前端和诊断接口不得返回密钥正文。

安装升级不得覆盖：

- `config/adapter.json`
- `run/provider_api_key`
- `run/provider_api_keys/`
- 工作流档案元数据

## 12. 异常处理

- 没有打开演示文稿：前端提示“请先打开演示文稿”。
- 无法取得当前页：前端提示“未能读取当前幻灯片”，不提交模型任务。
- 当前页内容不足且用户未填写要求：提示“请填写本页主题或生成要求”。
- 当前页读取超限：继续提交截断后的内容，并显示截断提示。
- adapter 暂时不可达：保留未完成任务号，按恢复策略继续查询。
- provider 认证失败或明确拒绝请求：任务进入失败态，并在设置页保留脱敏诊断。
- provider 返回非标准结果：展示最终原始回复，不显示空白结果区。
- 前端结构化结果渲染异常：降级为安全 Markdown 或纯文本展示。

所有异常路径都不得触发 PPT 写回。

## 13. 交付与测试

统一交付包同时包含：

- Word 插件 `wps-ai-assistant_1.0.0`。
- Excel 插件 `wps-ai-assistant-et_1.0.0`。
- PPT 插件 `wps-ai-assistant-wpp_1.0.0`。
- 更新后的 adapter、配置示例、安装脚本、卸载脚本、冒烟测试和 Dify 工作流手册。

自动化测试至少覆盖：

- Word、Excel、PPT Ribbon 和设置页宿主隔离。
- 当前页标题和文本形状清洗。
- 相邻页只读取标题。
- 输入预算、单形状截断和总量截断。
- `generate` 与 `optimize` 自动判定。
- 空白页且无用户要求时不提交任务。
- JSON、Markdown、`<think>` 和原始回复兜底解析。
- `clientJobId` 幂等、长任务轮询和恢复。
- 工作流档案按任务类型隔离。
- 安装升级保留现有配置和密钥。
- PPT 插件不存在任何幻灯片写回调用。
- 420x900 任务窗格下无文本、按钮或下拉菜单重叠。
- 现有 Word、Excel 测试全部继续通过。

目标机验收场景至少包括：

1. 有标题和正文的普通内容页优化。
2. 只有标题的页面按用户要求生成。
3. 完全空白页按用户要求生成。
4. 没有标题占位符、但有多个文本框的页面。
5. 仅图片或仅图表页面。
6. 当前页正文超出输入预算。
7. 模型普通模式返回。
8. 模型 think 模式慢返回。
9. 任务执行中关闭并重新打开任务窗格。
10. 新版本覆盖安装后，原有 Word、Excel、PPT 工作流配置仍存在。

## 14. 重点保护边界

- Word 和 Excel 现有 Ribbon、API、任务类型和工作流配置不得因 PPT 接入而改变。
- PPT 不读取整份演示文稿，不把全部幻灯片文本发送给 adapter 或模型。
- PPT 首版严格只读，不新增任何写回、占位符修改或版式操作。
- Word 智能编写现有写回逻辑不得抽取成 PPT 共用写回能力。
- 文档审查和 Excel 智能分析的长任务恢复行为不得回退。
- `/chat-messages` 新旧 Dify 入参兼容和任务级 API Key 选路不得改变。
- `<think>` 清理、Markdown 安全渲染、危险链接拦截和脱敏诊断必须继续生效。
- 统一安装包覆盖升级时不得清空现场配置和密钥。

## 15. 后续路线

PPT 单页内容助手稳定后，下一候选版本为 Excel 公式助手：基于选中表头、单元格和用户计算目标生成公式及说明，只预览和复制，不直接写入单元格。

Excel 双区域或双工作表对比不直接把原始数据交给模型。只有在本地统计、差异提取和摘要压缩方案验证后，才进入独立设计。
