# Word/Excel/PPT 克制型 Apple 界面体验设计

日期：2026-07-22

状态：已完成方案确认，待用户审阅书面设计

范围：Word、Excel、PPT 三个 WPS JS 插件的任务界面与设置界面

## 1. 背景

三宿主任务窗格已经具备统一的标题栏、宿主配色、工作流档案和设置入口，但设置体验仍存在以下问题：

- “模型接口 / 未配置”依赖隐藏的统一 Key 判定，不能反映当前宿主各任务当前工作流的真实就绪状态，状态刷新也不及时。
- “每项任务可保存多个工作流”长期占据设置页纵向空间。
- 没有备注的工作流仍显示“暂无备注”，增加无效信息。
- Word 使用任务选项卡，Excel 和 PPT 的工作流区域结构不同，不利于后续扩展真实功能。
- “联调状态”和“最后一次任务诊断”持续展开，占用紧凑任务窗格的主要可视空间。
- 三宿主的任务页和设置页虽然已有一致基础，但控件反馈、展开动效、焦点行为和状态稳定性仍缺少统一规则。

本设计采用“克制应用 Apple 设计原则”方案：使用系统字体、清晰层级、稳定空间关系、即时按压反馈和短促可中断动效，保持 Word 蓝、Excel 绿、PPT 橙不变。设计目标不是模仿 macOS，也不引入大面积毛玻璃、弹簧动画、拖拽手势或新的前端依赖。

## 2. 目标

1. 让模型接口状态准确、及时地反映当前宿主当前工作流的准备情况。
2. 统一 Word、Excel、PPT 的任务界面和设置界面的信息层级、控件行为及视觉节奏。
3. 在窄任务窗格中减少长期占位信息，通过提示层和渐进披露保留完整能力。
4. 为 Excel、PPT 后续新增真实功能预留可扩展选项卡结构，但当前只显示已交付功能。
5. 提升按钮、选项卡、提示层、折叠区和长任务状态的反馈清晰度与可访问性。
6. 严格保护现有模型请求、长任务、结果解析、企业知识、复制、预览和 Word 回写逻辑。

## 3. 已确认决策

1. 采用克制型 Apple 设计原则，三宿主现有配色保持不变。
2. 设置界面与任务界面使用同一视觉语言和交互反馈规则。
3. 模型接口就绪度由统一 API URL 与当前宿主各任务当前激活工作流共同决定，不再使用隐藏统一 Key 作为用户可见状态依据。
4. 当前宿主所有任务的当前工作流均有 Key 时显示“已就绪”；部分满足时显示“部分就绪”；没有任务满足或 URL 缺失时显示“未配置”；adapter 无法返回检测结果时显示“无法检测”。
5. 备用工作流、未激活工作流和兼容性统一 Key 回退不计入用户可见就绪度。
6. 设置页进入、URL 或工作流档案发生变更后立即刷新状态；设置页可见期间每 30 秒刷新，离开后停止。
7. 模型接口区域使用两行紧凑布局：标题与状态、截断 URL 与修改操作。
8. 多工作流说明改为信息图标提示，不再持续占据页面空间。
9. 没有备注的工作流完全不渲染备注行。
10. “联调状态”和“最后一次任务诊断”合并为默认折叠的“高级诊断”。
11. Excel 和 PPT 也显示任务选项卡，当前分别只显示“智能分析”和“智能总结”。
12. 所有宿主的任务选项卡使用单行横向滚动结构，只展示当前版本真实存在的功能。
13. 三宿主继续使用各自独立插件目录和前端运行时，不新增共享构建系统。

## 4. 设计原则

### 4.1 一致不等于完全相同

三宿主共享以下合同：

- 顶部标题、连接状态、设置与返回入口的位置和行为。
- 内容区纵向节奏、控件高度、圆角、边框、焦点和按压反馈。
- 任务选项卡、模型接口、工作流列表、编辑子页、信息提示和高级诊断的结构。
- 加载、成功、警告、错误和不可检测状态的中文语义。

以下差异必须保留：

- Word 使用蓝色，Excel 使用绿色，PPT 使用橙色。
- Word 只显示四类 Word 任务和企业知识入口。
- Excel 只显示已交付的智能分析，不显示企业知识。
- PPT 只显示已交付的智能总结，不显示企业知识。
- 各宿主的输入控件、结果操作和业务行为继续由现有任务能力决定。

### 4.2 克制型 Apple 原则

- 使用系统字体、清晰字号层级和中性内容背景。
- 圆角不超过 8px，卡片只用于真正独立的内容单元，不创建嵌套卡片。
- 宿主色只用于主操作、活动选项卡、焦点和少量强调，不大面积覆盖背景。
- 操作按下时立即反馈；展开和提示只使用短促的透明度、尺寸或位移动效。
- 不使用大面积 `backdrop-filter`、弹簧或回弹动画、拖拽切换、手势依赖和装饰性渐变。
- 所有动效必须支持 `prefers-reduced-motion`，减少动态效果时直接切换状态。

## 5. 统一界面架构

### 5.1 顶部外壳

任务页和设置页统一使用稳定的顶部区域：

1. 左侧显示“WPS AI 助理”和当前页面或功能名称。
2. 右侧显示设置或返回图标按钮。
3. 最右显示本地 adapter 连接状态，成功时使用“已连接”。

顶部区域保持稳定尺寸，不因状态文案、加载提示或页面切换发生跳动。任务内容独立滚动，顶部导航和连接状态保持可见。宿主连接状态与模型接口就绪度是两个不同概念：前者表示本地 adapter 是否可访问，后者表示当前宿主模型任务是否已完成必要配置。

### 5.2 任务界面

任务页按以下纵向路径组织：

1. 当前功能及工作流选择。
2. 任务输入与必要参数。
3. 当前要求或任务摘要。
4. 高对比度纯文字主操作按钮。
5. 稳定的任务状态与结果预览区域。

三宿主复用相同的间距、控件状态和结果区标题层级，但不强行统一不同业务的字段。主操作按钮继续只显示纯文字，不新增图标。处理中、完成或失败时在现有结果区域内更新状态，不销毁并重建整个结果容器，避免空白、闪烁和布局跳动。

Word 的预览、对照、纯文本、复制和回写能力保持现状；智能仿写不新增对照和回写。Excel、PPT 继续只提供预览与复制，不新增写回能力。

### 5.3 设置首页

设置首页按以下顺序组织：

1. 模型接口。
2. Word 独有的企业知识入口。
3. 工作流设置及任务选项卡。
4. 默认折叠的高级诊断。

页面采用单列、紧凑布局。各区域为平级内容单元，不在卡片内继续嵌套卡片。工作流新增和编辑继续进入全宽子页，避免在首页同时堆叠多个表单。

## 6. 模型接口状态

### 6.1 用户可见状态

| 状态 | 条件 | 用户文案 |
| --- | --- | --- |
| 已就绪 | API URL 已配置，当前宿主全部任务的当前激活工作流均有 Key | 已就绪 |
| 部分就绪 | API URL 已配置，当前宿主只有部分任务的当前激活工作流有 Key | 部分就绪 · `n/m` |
| 未配置 | API URL 缺失，或当前宿主没有任何任务的当前激活工作流有 Key | 未配置 |
| 无法检测 | adapter 无法返回配置状态或请求失败 | 无法检测 |

状态不执行真实模型请求，也不代表模型后台当前一定可用。用户可见状态不得因统一 Key 兼容回退而显示为就绪。

### 6.2 紧凑布局

模型接口区域固定为两行：

- 第一行：`模型接口`、状态徽标。
- 第二行：省略显示的 API URL、`修改`操作。

URL 缺失时显示“未配置接口地址”。URL 过长时单行省略，并通过可访问名称提供完整值。删除无实际作用的常驻刷新按钮；自动刷新承担日常同步，高级诊断保留人工刷新入口。

### 6.3 刷新机制

- 进入设置首页后立即刷新。
- API URL 保存成功后立即刷新。
- 工作流创建、编辑 Key、激活或删除成功后立即刷新。
- 设置首页可见期间每 30 秒刷新一次。
- 离开设置页、进入设置子页或任务窗格不可见时停止定时刷新。
- 每次请求分配递增序号，只有最新请求可以写入界面，防止慢响应覆盖新状态。
- 刷新期间保留上一次稳定状态并显示轻量检测反馈，不先清空为未知状态。

高级诊断不随模型接口状态每 30 秒轮询，只有用户展开或主动刷新时才请求。

## 7. 工作流设置

### 7.1 任务选项卡

- Word 显示智能编写、智能仿写、文档审查、格式审查。
- Excel 当前只显示智能分析。
- PPT 当前只显示智能总结。
- 不显示尚未开发的占位选项卡。

选项卡固定单行，不压缩文字。空间不足时横向滚动，左右边缘使用轻微遮罩提示仍有内容；活动项自动滚动到可视区域。支持指针、触控板和键盘方向键，`Home`、`End` 可跳至首尾。未来 Excel 或 PPT 新增真实功能时，只需向本宿主任务定义添加选项卡，不改变整体布局。

### 7.2 多工作流提示

“工作流设置”标题旁显示信息图标。提示层说明“每项任务可保存多个工作流，可在任务页选择当前使用的工作流”。提示层支持：

- 鼠标悬停。
- 键盘聚焦。
- 点击打开或关闭。
- `Esc`、点击外部和焦点离开关闭。

提示层锚定信息图标，不能遮挡当前工作流名称和主要操作。窄窗口下自动限制宽度并在视口内定位。

### 7.3 工作流列表

列表继续显示名称、必要状态和操作。备注为选填：

- 有备注时显示单行摘要，超出宽度省略。
- 无备注时完全省略备注节点和对应间距。
- 不再显示“暂无备注”。

当前、Key 未配置、读取失败等状态继续使用中文。新建、编辑、激活和删除规则沿用现有工作流档案设计，不改变接口和服务端保护。

## 8. 高级诊断

设置首页底部使用一个默认折叠的“高级诊断”披露区，折叠态只显示：

- 标题。
- 最近一次已知的简短状态摘要。
- 展开指示。

展开后显示现有联调状态和最后一次任务诊断，并提供刷新、复制操作。规则如下：

- 每次从折叠变为展开时自动刷新一次。
- 展开期间允许用户点击小型刷新图标再次刷新。
- 复制按钮只在展开后出现，并提供中文悬停说明和可访问名称。
- 重新进入设置首页时恢复折叠，避免长期占用空间。
- 诊断加载失败时保留已有内容并显示局部错误，不影响模型接口和工作流设置。

折叠和展开使用约 160ms 的透明度与尺寸过渡，不回弹、不过冲；减少动态效果模式下立即切换。

## 9. 任务状态与长任务体验

本设计不改变任何任务接口、超时、轮询、重试或后台 job 机制，只统一前端表现：

- 用户提交后立即进入稳定的“处理中”状态。
- 状态区域保留任务名称、当前阶段和等待说明，避免长时间空白。
- 状态更新只修改必要文本和进度提示，不重建整个结果区。
- 本地 adapter 不可访问、模型后台处理较慢、状态查询暂时失败和模型最终失败使用不同中文文案。
- 任务恢复继续使用既有 `clientJobId`、本地保存和恢复查询逻辑。
- Think 内容过滤、原始回复兜底、结果结构化解析和复制源文本保持不变。

任务页工作流切换、进入设置和返回任务页不得清空用户输入、现有结果、文档审查处理状态、未完成任务信息或 Word 回写按钮状态。

## 10. 动效、反馈与可访问性

### 10.1 即时反馈

- 按钮在指针按下时立即改变背景或产生不超过 `0.98` 的轻微缩放，持续约 100ms。
- 悬停不能引发布局变化。
- 页面级切换不使用整页滑动；只对局部内容使用 120–180ms 的透明度或轻微位移过渡。
- 禁用态保持稳定尺寸，并明确降低对比度和禁止重复提交。

### 10.2 键盘与焦点

- 所有按钮、选项卡、信息图标和披露控件均可使用键盘到达。
- 焦点样式使用当前宿主色，但错误状态仍使用独立错误色。
- 选项卡使用符合语义的 `tablist/tab/tabpanel` 关系。
- 披露控件同步维护 `aria-expanded` 和目标区域关系。
- 提示层和状态变化提供恰当的可访问名称；不把整页普通刷新作为打断式播报。

### 10.3 窄窗口规则

- 文字可换行，但按钮图标、状态徽标和固定格式控件不得被挤压变形。
- 最长工作流名称、错误文案和 URL 必须在容器内省略或换行。
- 固定格式的选项卡、按钮和徽标使用稳定尺寸，动态文本不得推动相邻控件发生明显位移。

## 11. 数据流与实现边界

三宿主继续调用现有 adapter 接口，不新增模型业务接口。前端使用现有配置和工作流档案数据计算当前宿主就绪度：

1. 读取 API URL。
2. 按当前宿主定义读取真实任务类型。
3. 找到每项任务当前激活工作流。
4. 判断当前工作流是否已配置 Key。
5. 计算已就绪任务数量和总任务数量。
6. 映射为已就绪、部分就绪或未配置。

计算逻辑应提取为可独立测试的纯函数，但继续保留在各宿主现有前端边界内。三宿主可采用相同函数合同和 DOM 标记，不建立新的跨插件运行时依赖。跨宿主一致性由契约测试保障。

## 12. 错误处理

- 模型接口状态读取失败：显示“无法检测”，保留已显示的 URL 和工作流内容。
- 工作流读取失败：显示局部错误和重试入口，不把失败解释为空列表。
- 配置保存部分成功：继续使用现有精确反馈，不笼统显示全部失败。
- 提示层定位失败：降级为图标下方的视口内文本层，不遮挡主要操作。
- 高级诊断读取失败：只影响诊断展开区，不改变模型接口状态。
- 定时刷新失败：保持上一次稳定内容并显示非阻塞错误，下一周期仍可重试。
- 页面切换时未完成的旧请求返回：通过请求序号忽略，不覆盖当前页面状态。

所有用户可见反馈优先使用中文，不显示 `Dify`、`OK`、内部路由名或技术异常堆栈。

## 13. 兼容与保护边界

- 不修改智能编写、智能仿写、文档审查和格式审查的业务逻辑。
- 不修改智能编写预览、对照、复制、格式保持和 Word 回写。
- 不修改文档审查处理状态、审查记录、长任务恢复和结果兜底。
- 不修改格式审查识别、模板规则或本地回退。
- 不修改 Excel 智能分析和 PPT 智能总结的只读边界。
- 不修改 PPT 文件上传、Dify 文件引用和同一认证快照规则。
- 不修改模型等待预算、轮询容错、Think 内容剥离或新旧 Dify 输入兼容。
- 不修改企业知识匹配、导入导出、备份、诊断或 fail-open 行为。
- 不修改 Ribbon、manifest、adapter、安装脚本和覆盖安装配置保护。
- 不新增依赖，不大规模重构三个插件目录。

## 14. 测试策略

### 14.1 单元与行为测试

- 就绪度纯函数覆盖 URL 缺失、全部就绪、部分就绪、全部未就绪和无法检测。
- 只统计每项任务当前激活工作流，不统计备用档案和统一 Key 回退。
- 进入设置、URL 保存、工作流变更后立即刷新。
- 设置页可见时 30 秒刷新，离开或隐藏后停止。
- 使用伪计时器验证不重复建立定时器。
- 慢的旧请求不能覆盖新的状态。
- 没有备注时不渲染备注节点或“暂无备注”。
- 信息提示支持悬停、聚焦、点击、`Esc` 和点击外部关闭。
- 高级诊断默认折叠、每次展开刷新、允许手动刷新和复制、重新进入复位。
- 任务页和设置页切换不清空输入、结果、任务恢复信息或回写状态。

### 14.2 跨宿主契约测试

- 三宿主具有相同顶部结构、连接状态中文映射、任务选项卡语义和高级诊断结构。
- Word 只显示四类 Word 任务和企业知识。
- Excel 只显示智能分析，不显示企业知识。
- PPT 只显示智能总结，不显示企业知识。
- 三宿主只读取本宿主任务类型，不出现交叉工作流。
- 宿主色仍分别为蓝、绿、橙；成功、警告、错误色不被宿主色替代。
- 三个主生成按钮仍为高对比度纯文字按钮。

### 14.3 回归测试

运行现有工作流设置、布局、长任务、企业知识、结果预览和宿主辅助函数测试，并对三个 `taskpane.js` 执行语法检查。测试失败不得通过删除旧断言或放宽业务契约规避。

### 14.4 视觉验收

在 `420x900` 和 `320x700` 两种视口分别检查 Word、Excel、PPT：

- 任务页和设置页的默认状态。
- 模型接口已就绪、部分就绪、未配置和无法检测。
- 单工作流、多工作流、无备注、Key 缺失和读取失败。
- 信息提示打开、诊断展开和选项卡溢出。
- 处理中、成功、状态查询暂时失败和最终错误。
- 键盘导航、焦点样式和减少动态效果模式。

验收不得出现文字遮挡、按钮位移、无意义空白、嵌套卡片、横向页面滚动或宿主间功能交叉。

## 15. 风险与控制

- **状态语义改变风险**：以独立纯函数和契约测试固定就绪度规则，明确区分 adapter 连接与模型接口就绪度。
- **轮询增加负担风险**：仅设置首页可见时每 30 秒刷新轻量配置状态，高级诊断不进入轮询。
- **旧请求覆盖风险**：所有刷新使用请求序号，仅最新请求可提交界面状态。
- **窄窗口拥挤风险**：使用两行模型接口、单行滚动选项卡、无备注省略和诊断折叠，并在 320px 宽度验收。
- **视觉改动影响业务风险**：采用宿主局部修改和跨宿主契约测试，不抽取新的共享运行时，不改业务函数入口。
- **Linux WebView 性能风险**：不使用大面积模糊、复杂阴影、弹簧库和持续动画，只使用短促 CSS 过渡。
- **任务状态丢失风险**：设置切换继续复用现有任务 DOM 和状态对象，不重新初始化业务页面。

## 16. 非目标

- 不新增 Word、Excel、PPT 业务功能。
- 不为尚未开发的功能显示选项卡或入口。
- 不增加统一任务中心、历史记录或跨宿主工作流管理。
- 不调整模型提示词、温度、模型选择或输出格式。
- 不修改 API URL、Key、工作流档案或企业知识的数据结构。
- 不重新设计 Ribbon，也不制作新图标。
- 不在本设计阶段修改版本号、README、交付包或安装包。

## 17. 验收标准

1. 三宿主任务页和设置页具有一致的信息层级、控件反馈和紧凑布局。
2. Word、Excel、PPT 分别保持蓝、绿、橙宿主配色。
3. 模型接口状态准确反映当前宿主各任务当前激活工作流，并能自动更新。
4. 无备注工作流不显示“暂无备注”，多工作流说明不再常驻占位。
5. 高级诊断默认折叠，展开后可刷新、复制且不影响其他设置。
6. Excel 和 PPT 使用与 Word 一致、可扩展但只显示真实功能的任务选项卡。
7. 任务长时间运行时界面持续提供稳定反馈，不出现无反馈或状态区域闪烁。
8. 任务页、设置页和子页往返不丢失任务输入、结果、恢复信息或回写状态。
9. 自动测试、JavaScript 语法检查和两种窄视口视觉验收全部通过。
10. adapter、模型请求、长任务、结果解析、企业知识、复制和回写行为没有变化。
