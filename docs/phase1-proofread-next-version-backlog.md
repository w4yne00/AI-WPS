# 格式校对后续版本备忘

更新时间：2026-05-09

本文件记录一期格式校对“可闭环版本”暂不实现、但下一版本应继续规划的能力。

## 下一版本候选能力

- 页眉页脚抽取与合规检查：识别页眉文本、页脚文本、页眉页脚位置、是否符合模板要求。
- 页码抽取与合规检查：识别页码位置、格式、起始页、是否缺页或重复。
- 表格深度抽取：抽取表格行列、表头、单元格样式、表格标题、跨页表头、表格编号。
- 图片与图题抽取：识别图片、图题、图号连续性、图题格式和引用关系。
- 文档内高亮定位：根据审校结果在 WPS 文档中定位段落并高亮问题片段。
- 自动批量修复：对确定性格式问题支持批量修复，并提供修复前预览和撤销策略。
- 多模板 RAG 检索配置界面：支持选择模板知识库、模板版本、行业/文档类型元数据过滤。
- 当前章节检查范围：在“全文 / 选中文本”之外增加“当前章节”，适合技术文档长文分段审校。
- 审校结果交互增强：按问题类型筛选、按严重程度排序、点击问题定位、导出审校报告。

## Provider 与 Dify 工作流演进备忘

一期闭环采用“单 provider + 单 API Key + 单 Dify 工作流 + task_id 判断节点”。下一版本需要根据目标机和 Dify 实际使用情况，继续评估是否升级为“单 provider + taskRoutes + 多任务密钥 / 多工作流”。

候选能力：

- `taskRoutes` 完整配置模型：为每个任务增加 `path`、`apiKeyRef`、`responseMode`、`workflowId`、`enabled`、`fallbackToDefault`。
- 多任务密钥存储：按任务保存密钥文件，例如 `word_proofread.key`、`word_technical_review.key`，继续禁止前端保存密钥。
- 任务级联调状态：设置页显示每个任务的配置状态、最近一次调用状态和错误提示。
- Dify 多工作流拆分：将格式校对、技术审查、智能改写/续写、智能排版拆成独立 Dify 工作流，降低单工作流复杂度。
- 工作流版本管理：记录每个任务当前使用的工作流版本、模板版本和提示词版本，便于验收和回滚。
- 任务级降级策略：某个 Dify 任务不可用时，允许该任务回退 mock 或本地规则，其它任务不受影响。
- 设置页高级路由配置：默认隐藏，仅管理员或实施人员需要时打开，避免干扰普通用户。
- 审计日志增强：记录 `task_id`、trace id、模板 id、工作流版本、provider 来源和调用耗时。

## 当前版本边界

当前一期增强只做可闭环能力：

- WPS 插件抽取段落、标题和基础样式结构。
- adapter 接收 `documentStructure`。
- 本地规则检查格式合规。
- Dify/企业大模型检查错别字、语病、表述不规范、逻辑不清、章节命名不统一。
- WPS 任务窗格按分类展示问题清单。

页眉页脚、页码、表格深度抽取、高亮定位、自动批量修复、多模板 RAG 配置和当前章节范围留到下一版本。
