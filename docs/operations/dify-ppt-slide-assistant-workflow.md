# PPT 单页助手 Dify 工作流配置

## 接口与任务

- 任务类型：`ppt.slide_assistant`
- 模型接口：Dify `/chat-messages`
- adapter 接口：`POST /ppt/slide-assistant/jobs`、`GET /ppt/slide-assistant/jobs/{jobId}`
- provider 等待预算：1800 秒

在 PPT 任务窗格设置页创建具名工作流档案并保存对应 API Key。切换档案只影响下一次新任务，不改变已经提交的后台任务。

## 输入范围与预算

adapter 只接收当前页主标题、可选副标题、当前页普通文本形状、前一页标题、后一页标题和用户补充要求。动态输入总预算为 4600 字符：主标题 200、副标题与正文合计 3000、相邻页标题各 200、补充要求 1000；单个正文文本形状最多 1000 字符。

正文达到 20 个非空白字符时使用 `optimize`，不足时使用 `generate`。主标题和副标题不参与阈值计算；生成模式必须提供补充要求。

## 推荐输出

```markdown
## 建议标题
项目总体进展

## 核心要点
- 总体方案设计已完成
- 系统进入联调阶段
- 重点关注接口稳定性

## 本页结论
项目按计划推进，下一阶段应集中完成接口联调和风险收敛。
```

也可返回包含 `suggestedTitle`、`bullets`、`conclusion` 和可选 `plainText` 的 JSON。adapter 会先剥离 `<think>...</think>` 内容，再依次解析 JSON、固定 Markdown 分区；无法结构化解析时保留最终原始回复供前端显示。

## 模型约束

工作流提示词必须要求模型：

1. 只输出建议标题、3 至 5 条核心要点和一句本页结论。
2. 不编造当前页和用户补充要求中没有的事实或数据。
3. 不扩写相邻页面，相邻标题只用于衔接和避免重复。
4. 不输出配色、版式、图片、动画或页面操作建议。
5. 不声称已经修改 PPT，不输出深度思考过程。

PPT 插件首版只预览和复制结果，不写回幻灯片。
