# AI-WPS 工作流配置档案管理

## 适用范围

`v0.16.0-alpha` 起，Word 和 Excel 功能可保存多个具名 Dify 工作流 API Key；`v0.17.0-alpha` 起，PPT 功能使用同一套档案机制；`v0.18.0-alpha` 的当前页总结和文档总结继续共用一个 PPT 档案：

- Word：智能编写、智能仿写、文档审查、格式审查。
- Excel：智能分析，内部任务键 `excel.analysis`。
- PPT：智能总结，内部任务键 `ppt.slide_assistant`；当前页总结和文档总结共用同一档案。

Word、Excel 和 PPT 共用本地 adapter 配置，但前台只显示当前宿主对应的工作流，不交叉显示。

## 新增工作流档案

1. 打开 WPS AI 助理的“设置”。
2. 找到“工作流配置管理”中的目标功能。
3. 输入自定义名称、API Key 和可选备注。
4. 如需立即使用，勾选“保存后设为当前”。
5. 点击“保存工作流”。

建议名称包含用途或版本，例如“文档审查稳定版”“表格分析测试版”“PPT 智能总结汇报版”。同一功能内名称不能重复。

## 切换当前工作流

功能页面会显示“当前工作流”下拉菜单。选择目标档案后点击“切换”，界面提示成功后，从下一次新任务开始使用该档案。已经提交的文档审查、智能分析或智能总结后台任务继续使用提交时的密钥，不会被中途切换影响。

文档总结在一次任务内固定使用提交时解析出的同一份认证快照：先调用 Dify `/files/upload`，再携带返回的文件引用调用 `/chat-messages`。任务运行中切换档案不得改变这两个请求的 API Key，也不得触发重复上传或重复提交。

设置页也可通过“设为当前”完成同样操作。密钥缺失的档案不能激活。

## 修改和删除

- “保存名称”只更新自定义名称和备注，不影响密钥。
- “更换密钥”要求重新输入完整 API Key，保存后输入框立即清空。
- 当前使用的档案不能直接删除。先切换到其他档案，再删除旧档案。
- 删除备用档案会同时删除它对应的独立密钥文件，无法恢复。

## 旧配置迁移

升级后，原 `taskApiKeyRefs` 中已有的任务级密钥会自动显示为“当前配置”。迁移复用原密钥文件，不移动、不覆盖、不删除旧密钥。每次激活新档案时，adapter 同步更新旧 `taskApiKeyRefs` 映射，便于旧前端或回退版本继续使用最后一次选择。

## 存储与安全

- 档案名称、任务类型、备注和密钥引用保存在 `config/adapter.json`。
- API Key 正文保存在 `run/provider_api_keys/`，不会通过档案查询接口返回。
- 新建密钥文件权限为 `0600`。
- 最近一次任务诊断只显示档案名称、档案 ID、密钥引用和认证来源，不显示密钥正文。
- 安装程序覆盖升级时继续备份并恢复 `config/adapter.json`、`run/provider_api_key` 和整个 `run/provider_api_keys/`。

## 故障排查

若切换失败：

1. 确认目标档案显示“密钥已配置”。
2. 在设置页重新输入该档案的完整 API Key。
3. 刷新配置后重新切换。
4. 执行一次任务，在“最近一次任务诊断”确认 `workflowProfileName` 和 `taskApiKeyRef` 是否对应目标档案。

若工作流档案列表无法加载，已有任务仍沿用 adapter 当前激活的密钥；列表恢复前不要重复保存同名档案。
